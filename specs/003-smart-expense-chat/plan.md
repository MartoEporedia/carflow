# Implementation Plan: Smart Expense Chat Input

**Branch**: `003-smart-expense-chat` | **Date**: 2026-04-30 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `specs/003-smart-expense-chat/spec.md`

## Summary

Add a dedicated chat-style screen that allows users to log car expenses by typing a natural language message or attaching a photo of a receipt/invoice. The app parses the input (via the existing LLM/parser pipeline), identifies missing required fields, and asks one follow-up question at a time until the expense is complete. The user reviews a confirmation summary and saves it. The feature is implemented as a new navigation route; all text input continues to route through the existing NLP pipeline; photo analysis is a new multimodal LLM call added to the `shared:network` module.

## Technical Context

**Language/Version**: Kotlin 1.9.22, Java 17 bytecode  
**Primary Dependencies**: Jetpack Compose + Material3 BOM 2024.01.00, Hilt 2.50, Room 2.6.1, Kotlinx Serialization 1.6.2, Ktor (existing in `shared:network`)  
**Storage**: Room 2.6.1 (existing schema — no migration required); chat session is in-memory only  
**Testing**: JUnit 4 + Kotlin coroutines test for ViewModel; Compose UI tests for screens  
**Target Platform**: Android API 26+ (minSdk 26)  
**Project Type**: Android mobile app (MVVM + Repository + Hilt DI)  
**Performance Goals**: LLM round-trip ≤ 3 s on good connectivity; local follow-up resolution < 50 ms  
**Constraints**: Offline-capable for text input (rule-based fallback); photo analysis requires network; no WebSocket; no persistent chat storage  
**Scale/Scope**: Single user, single active session at a time; ephemeral in-memory state
